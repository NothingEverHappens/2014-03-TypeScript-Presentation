Why TypeScript?

Features

  • Superset of Javascript: Any JS is a valid TS
  • Support for some ES6 features
  • Works seamlessly with JS libraries
  • Static type checking

Tools Support

  • Plugins for IDEs: Intellij, Eclipse, Sublime, Visual Studio
  • Plugins for Build Tools: Grunt, Gulp, Broccoli
  • TypeScript compiler is an npm module
  • Typescript.api for parsing and getting AST

Creators

  • Created by Anders Hejlsberg (C#, Turbo Pascal)
  • Supported by Microsoft

Compiling to JavaScript

  • Can be compiled on save
  • Incremental compilation
  • Can be compiled to ES3 or ES5
  • Really clean output

Static Typing

Simple types

var s: string = "Hello";
var c: number = 121;
var r: boolean = true;
var t: Number[]  = [1,2,3];
var i: any; 
var p: void = undefined;

Complex types

var point:{ 
    x: number,
    y: number
};
point = { x: 0, y: 1}

Interfaces

interface IPoint {
  x:number;
  y:number;
  z:number;
}
 
var point:IPoint;

Function types

var stringLength:(string:String) => number;

Generics

function identity<T>( x:T ){ 
    return x
};

var x:number = identity("3");

Type definitions for type information for other libraries

  • Can be downloaded from github.com/borisyankov/DefinitelyTyped
  • Or can be installed via TypeDefinition Package manager tsdpm.com

Modules

Modules

  • TypeScript modules can be compiled into AMD or common JS
  • AMD Needs some special care as TS doesn't understand require.js config
  • TypeScript Module compiler can be used to create smaller submodules

Testing

Testing

  • Same libraries that work for JS will work for TS
  • TS Unit
  • Karma TypeScript Preprocessor

Debugging

  • TypeScript has source maps enabled by default
